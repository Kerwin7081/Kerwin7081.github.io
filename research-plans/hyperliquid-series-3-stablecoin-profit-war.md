# 二、全局核心判断

## 判断一：稳定币行业已从“发行规模竞争”进入“利润分配权竞争”

过去的模式是：

> 发行人创造稳定币，渠道帮助发行人分销。

正在形成的新模式是：

> 渠道掌握用户、交易、支付和钱包入口，稳定币发行人竞争成为渠道的结算货币。

Hyperliquid 的 AQAv2、Circle 与 Coinbase 的分销关系、Bridge Open Issuance，以及 Open Standard 推出的 Open USD，均指向同一趋势：

> **谁创造真实使用场景，谁就要求分享储备收益。**

## 判断二：Bridge 正在从两个方向攻击传统稳定币发行模式

必须区分两套相关但不同的机制：

### 1. Open Issuance（开放发行）

由 Stripe 旗下 Bridge 提供的稳定币发行平台：

- 企业可以发行自己的品牌稳定币；
- Bridge 负责储备管理、合规、安全、铸造与赎回；
- 企业可选择储备结构并决定如何使用储备收益；
- 平台内不同品牌稳定币可以一比一互换，接入 Shared Liquidity Network（共享流动性网络）。

它解决的是：

> **企业为什么必须替 Circle 或 Tether 建立品牌和余额？**

### 2. Open Standard / Open USD（开放标准／开放美元）

Open Standard 是独立治理实体，参与方超过 140 家，首个产品是 Open USD（OUSD）：

- 没有单一机构控制；
- 企业可以免费铸造和赎回，不设人为规模上限；
- 大部分储备收益在扣除小额管理费后返还推动采用的参与方；
- 用于支付、交易、跨境资金流和 Agentic Commerce（智能体商业）。

它解决的是：

> **如果每家公司都发行一枚币造成碎片化，能否共同使用一枚中立结算币，同时让采用者分享利润？**

### Bridge 的真正战略

Bridge 并不是只做一种稳定币，而是在建立三层控制力：

1. **发行层**：通过 Open Issuance 帮助企业发行自己的币；
2. **共享结算层**：通过 Open Standard / OUSD 建立中立共同币；
3. **路由层**：无论最终使用哪一种币，Bridge 都掌握法币出入金、储备管理、转换与清算编排。

因此，Bridge 对 Circle 的威胁不只是多出一个竞争币，而是：

> **它试图把稳定币发行从品牌垄断业务，变成一个由基础设施平台控制的开放网络。**

## 判断三：稳定币强化的是“美元网络”，但不等于自动解决美国融资问题

稳定币把美元变成 Internet-native money（互联网原生货币），扩大海外个人、企业与机器对美元结算资产的使用。

但要区分三件事：

1. **交易与支付美元化**：更多交易用美元计价和结算；
2. **储备货币地位**：中央银行和大型机构持有美元资产；
3. **美国国债需求**：稳定币储备具体购买哪些期限的美国国债。

稳定币主要强化第一项，并通过储备资产间接影响第三项；它不能单独创造支撑美元储备货币地位所需的法治、金融市场深度、货币政策信誉和地缘政治能力。

## 判断四：Agent 经济可能显著增加稳定币交易量，但不一定同比例增加发行人利润

Agent 适合使用稳定币，因为稳定币支持：

- 7×24 小时结算；
- API 原生调用；
- 按次、按秒、按 Token 的微支付；
- 全球转账；
- 可编程限额和钱包授权；
- 机器对机器自动结算。

但稳定币发行人赚的是平均余额带来的储备收益，而不是单纯的交易次数。

因此必须区分：

> **Agent 经济可能先提高 Stablecoin Velocity（稳定币周转速度），只有当 Agent 需要预存工作资金、持有经营现金和累积收入时，才会显著提高 Stablecoin Float（稳定币沉淀余额）。**

# 四、全文核心公式

## 1. 毛储备收益

`毛储备收益 = 平均流通量 × 储备资产收益率`

## 2. 净储备经济

`净储备经济 = 毛储备收益 − 渠道分成 − 用户奖励 − 资产管理与托管成本 − 链上成本 − 合规与运营成本`

## 3. 单位稳定币净收益

`单位净收益率 = 净储备经济 ÷ 平均流通量`

## 4. Agent 经济对稳定币余额的影响

`Agent 平均稳定币余额 = Agent 数量 × 单个 Agent 预存工作资金 × 平均资金停留时间系数`

## 5. 稳定币对美债的净新增需求

`净新增美债需求 ≠ 稳定币市值增长`

更准确地说：

`净新增美债需求 = 来自非美元现金／未银行化资金的流入 + 新增经营现金 − 从货币基金和银行体系迁移后被替代的原有美债需求`

全文反复强调：

> 流通量决定收入基数，分销结构决定利润归属；交易量决定网络价值，但平均余额才决定储备收益。

# Part I｜一枚数字美元的利润表

## 01｜一枚数字美元为什么能够赚钱

用传统金融语言解释稳定币资产负债表：

| 稳定币结构 | 传统金融经济类比 |
|---|---|
| 用户交付 1 美元 | 形成发行人的低成本交易性负债 |
| 发行 1 枚稳定币 | 对用户形成可赎回数字负债 |
| 储备资产 | 现金、短期美债、回购和货币市场基金 |
| 储备收益 | Float Income（浮存金收益），近似利差收入 |
| 区块链 | 可编程清算与转账网络 |
| 钱包、交易所、支付 App | 分销渠道与客户入口 |

醒目标注：

- 稳定币在法律上通常不是银行存款；
- 通常不享受存款保险；
- “类似无息负债”只是经济类比，不是法律定性。

### 视觉一：一枚 USDC 的资产负债表

左侧：

- 1 美元稳定币负债

右侧：

- 现金
- T-bills（美国短期国库券）
- Repo（回购）
- Money Market Fund（货币市场基金）

下方：

- 储备收益流向发行人、渠道、用户和协议。

## 02｜一美元利润瀑布

展示一美元稳定币从毛收益到净收入的全过程：

1. 美国短端利率形成毛收益；
2. 资产管理机构收取管理费；
3. 银行与托管机构收取费用；
4. 发行人承担合规、技术和赎回责任；
5. Coinbase、Binance、Hyperliquid 等渠道要求分成；
6. PayPal、钱包或平台向用户支付奖励；
7. 剩余部分才成为发行人的净经济价值。

### 交互模块 A：利润瀑布计算器

输入：

- 平均余额
- 储备收益率
- 管理与托管成本
- 渠道分成比例
- 用户奖励比例
- 发行人运营成本

输出：

- 毛储备收益
- 渠道所得
- 用户奖励
- 发行人净收入
- 单位稳定币净收益率

默认案例：

- 余额：50 亿美元
- 储备收益率：3.5%
- 毛收益：1.75 亿美元
- 渠道／协议分成：90%（仅作 AQAv2 量级演示，实际按成本调整后口径）

# Part II｜谁在重新分配利润

## 03｜Hyperliquid：交易场景如何反向抽取稳定币利润

本节承接前两篇，不重复解释交易所机制。

### 核心问题

- Hyperliquid 为什么能要求约 90% 的成本调整后储备收益？
- Circle 为什么仍愿意合作？
- Coinbase 为什么担任资金管理部署方？
- Assistance Fund 如何成为协议价值捕获工具？
- 降息后高比例分润是否仍可持续？

### 核心结论

> Hyperliquid 没有发行美元，却凭借掌握高频交易、保证金和结算场景，取得了类似大型存款渠道的议价权。

### 必须澄清“90%”

- 不是储备本金的 90%；
- 不是 Circle 全公司收入的 90%；
- 不是未经扣除成本的毛利息；
- 是 Hyperliquid 相关供应量产生的成本调整后储备收益；
- 实际金额取决于平均余额、利率、成本、实施日期和结算规则。

### 视觉二：50 亿美元 USDC 利润分配图

用户 → USDC 保证金 → 储备资产 → 毛收益 → Circle / Coinbase / Hyperliquid Assistance Fund

## 04｜Circle 与 Coinbase：发行人与超级分销商

### Circle 的角色

- 发行与赎回；
- 储备管理与合规；
- 多链技术部署；
- 承担品牌、监管和流动性责任。

重点指标：

- 储备收入占比；
- 平均 USDC 流通量；
- 储备回报率；
- RLDC Margin；
- 分销成本；
- 非储备收入占比。

### Coinbase 的角色

- USDC 超级分销商；
- 平台余额持有人；
- 托管与法币出入金平台；
- Base 生态入口；
- Hyperliquid 资金管理部署方；
- Custom Stablecoins 基础设施商。

### 核心结论

> Circle 掌握发行资格和储备资产，Coinbase 掌握客户、交易、钱包和分销。稳定币利润由两者共同创造，也被两者共同分配。

## 05｜Tether 与 Circle：垂直整合和开放分销

| 维度 | Tether | Circle |
|---|---|---|
| 商业结构 | 高度垂直整合 | 开放分销网络 |
| 核心市场 | 离岸、交易、跨境美元 | 美国、机构、合规链上金融 |
| 利润留存 | 较高 | 受渠道分成影响显著 |
| 信息披露 | 定期鉴证 | 上市公司财报与 SEC 披露 |
| 核心风险 | 监管、透明度、司法管辖 | 降息、分销成本、渠道议价 |

核心问题：

> 市场最终奖励的是最高流通量、最高单位净收益，还是最强的全球分销网络？

## 06｜Bridge 的双重开放战略：Open Issuance 与 Open USD

这是第三篇新增的重点章节。

### A. Open Issuance：让每个平台拥有自己的币

- 企业拥有品牌和产品体验；
- 企业参与储备收益；
- Bridge 负责发行后台、储备、合规和流动性；
- 不同品牌币接入共享流动性并可一比一转换；
- Phantom CASH、USDH、MetaMask 等可作为案例。

### B. Open Standard / Open USD：让所有平台共享一枚中立币

- 由独立实体治理，不由单一发行人控制；
- 140 多家银行、支付、科技、加密和商业平台参与；
- 免费铸造和赎回；
- 大部分储备收益扣除管理费后返还推动采用的参与者；
- Visa Stablecoin Platform 首先接入 Open USD；
- 明确把 Agentic Commerce 列为使用场景。

### C. 两套机制为什么并不矛盾

| 机制 | Open Issuance | Open USD |
|---|---|---|
| 产品形态 | 多枚品牌稳定币 | 一枚共享稳定币 |
| 核心诉求 | 品牌、产品控制和自有经济 | 中立治理、统一流动性和共享收益 |
| 主要客户 | 钱包、企业、银行、金融科技 | 大型支付、银行、商业和交易网络 |
| Bridge价值 | 发行、储备和路由基础设施 | 参与共同标准并连接支付网络 |

### 核心结论

> Open Issuance 把发行权交给企业；Open USD 把储备利润交给分销网络。两者共同削弱传统单一发行人对品牌、流动性和储备收益的垄断。

### 视觉三：稳定币开放网络图

上层：企业品牌币 / OUSD 共享币  
中层：Bridge 编排、Privy 钱包、Visa 卡网络、Stripe 支付  
底层：储备管理、银行、货币基金、美债与区块链清算

## 07｜PayPal、Visa 与支付闭环

### PayPal / PYUSD

储备收益可以转化为：

- 用户奖励；
- PayPal / Venmo 余额留存；
- 商户支付补贴；
- 跨境汇款效率；
- 平台获客成本。

### Visa

Visa 不必成为最大稳定币发行人，也可以控制：

- 钱包入口；
- 商户接受网络；
- 稳定币结算；
- 铸造和赎回连接；
- 风控、身份与争议处理；
- Agent 支付授权。

### 核心结论

> 支付平台未必追求独占储备收益；它们可以把稳定币作为降低结算成本、增加交易量和保卫支付网络的工具。

## 08｜GENIUS Act 与收益转移

重点不是法律综述，而是商业模式：

- 支付稳定币发行人直接向持有人付息受到限制；
- 收益竞争可能转移到交易所、钱包、支付奖励、关联产品和代币化货币基金；
- 渠道方因此获得更高议价能力；
- 支付稳定币、收益型代币和 Tokenized Money Market Fund（代币化货币市场基金）的边界更加重要。

醒目标注：本节是商业模式与监管结构分析，不构成法律意见。

# Part III｜稳定币、美元结算权与美债需求

## 09｜稳定币是否强化美元结算霸权

专业表达建议：

> “美元结算霸权”在正文首次出现时注释为：美元在全球计价、支付、融资和安全资产网络中的结构性优势。

### 强化机制

1. **扩大美元可得性**  
   海外用户无需直接进入美国银行体系，也能持有和转移美元计价资产。

2. **降低美元结算摩擦**  
   稳定币可以 7×24 小时跨境转移，减少银行营业时间、代理行和地区限制。

3. **把美元嵌入互联网协议**  
   美元从银行账户中的余额，变成钱包、API、智能合约和 Agent 可调用的结算资产。

4. **扩大美元计价网络效应**  
   当更多交易、API、商品、工资和跨境资金使用美元稳定币时，其他参与者也更倾向使用美元。

5. **延伸美国监管与金融影响力**  
   合规发行人、托管机构和储备银行可以执行制裁、冻结、反洗钱和身份要求。

### 必须保留的边界

稳定币不能单独创造美元霸权。美元国际地位仍依赖：

- 美国法治与产权保护；
- 美联储信誉；
- 深度和流动性充足的资本市场；
- 美国经济体量；
- 国债作为全球安全资产的能力；
- 地缘政治与军事联盟。

因此结论应是：

> **稳定币是美元网络的数字分发层和结算加速器，而不是美元国际地位的唯一基础。**

### 对新兴市场的双面影响

正面：

- 提供低成本美元储值与跨境支付；
- 降低汇款费用；
- 改善高通胀国家居民的资产选择。

负面：

- 加快民间美元化；
- 削弱本地银行存款与信贷基础；
- 降低资本管制有效性；
- 增加本国货币政策传导难度；
- 把私人稳定币发行人变成事实上的跨境货币基础设施。

## 10｜稳定币怎样改变美国国债买方结构

### 已出现的结构变化

稳定币发行人正在成为新的 Non-traditional Treasury Buyer（非传统美债买方）：

- 主要购买美国短期国库券、现金和政府货币基金；
- BIS 估算，2025 年稳定币发行人购买约 350 亿美元美国国库券；
- 截至 2025 年底，稳定币管理资产超过 2,700 亿美元；
- 美国财政部与 TBAC 已把稳定币列入未来美债需求的重要观察变量。

### 但“稳定币增长 = 新增美债需求”并不成立

必须拆分资金来源：

#### 情景 A：资金来自海外现金、非美元资产或未银行化用户

- 形成更接近净新增的美元需求；
- 提高短期美债需求；
- 强化美元化；
- 对美国融资成本具有边际支持。

#### 情景 B：资金原本放在美元货币市场基金

- 只是从货币基金份额转为稳定币负债；
- 底层资产仍可能是同一批 T-bills；
- 美债净新增需求有限；
- 主要变化是持有结构和收益分配。

#### 情景 C：资金从商业银行存款迁移

- 稳定币发行人增加 T-bill 持有；
- 银行失去低成本存款；
- 银行可能减少贷款、证券投资或批发融资；
- 对美债和实体信用的净影响存在不确定性。

### 期限结构的关键限制

> 稳定币主要需要高流动性的短期资产，因此它们更可能成为 T-bills 的重要买方，而不是 10 年、30 年长期国债的主要接盘者。

这意味着：

- 能支持财政部短端融资；
- 不能直接解决长期财政赤字对应的久期供给；
- 可能进一步增加短端需求与长端供给之间的不平衡；
- 财政部可能根据结构性需求变化调整 Bills、FRNs 和 Coupons 的发行组合。

### 风险

- 大规模赎回造成短债和货币基金集中变现；
- 稳定币余额对市场信心和监管变化敏感；
- 需求集中在少数发行人与托管机构；
- 高利率推动发行增长，降息可能削弱商业激励；
- 私人稳定币可能把部分货币创造和美元分发权转移到非银行机构。

### 视觉四：美国短债买方结构变化

传统买方：

- 货币市场基金
- 银行
- 外国央行
- 外国私人投资者
- 企业现金

新增／扩大的买方：

- 稳定币发行人
- 代币化货币基金
- 链上协议储备
- Agent 钱包与平台经营现金

# Part IV｜Agent 经济与稳定币

## 11｜为什么 Agent 天然需要可编程货币

当前支付系统主要为人设计：

- 账户注册；
- 网页结账；
- 输入卡号；
- 订阅套餐；
- 人工确认；
- 退款与争议处理。

Agent 需要的是：

- API 可调用；
- 小额和高频；
- 自动报价与自动付款；
- 机器可读的支付条件；
- 可编程预算；
- 交易后立即获得数据、算力或服务。

### 已出现的协议层

- x402：Coinbase 推动的 HTTP 原生支付协议，常用 USDC 结算；
- MPP / Machine Payments Protocol（机器支付协议）：Stripe 与 Tempo 共同推出，支持稳定币、银行卡和其他支付方式；
- Agent Wallet（智能体钱包）：为 Agent 设置余额、限额、白名单和签名权限；
- Shared Payment Token（共享支付令牌）：让 Agent 使用人类已有支付方式，但不暴露原始凭证；
- MCP / UCP：用于工具发现、服务调用和商业流程协同。

### 重要判断

> Agent 支付不会全部转向稳定币。消费者购物可能继续依靠银行卡和支付令牌；API、算力、数据、跨境 B2B 和机器间微支付更适合稳定币。

## 12｜Agent 支付技术栈

### 六层结构

1. **Discovery（发现）**  
   Agent 找到商品、API、数据或其他 Agent。

2. **Quotation（报价）**  
   服务方返回机器可读价格和支付条件。

3. **Policy / Authorization（策略与授权）**  
   检查预算、限额、白名单、身份和用途。

4. **Payment Protocol（支付协议）**  
   x402、MPP、卡网络令牌或银行接口。

5. **Settlement Asset（结算资产）**  
   USDC、OUSD、其他稳定币、代币化存款或法币。

6. **Ledger and Compliance（账本与合规）**  
   区块链、银行账本、税务记录、AML/KYC 和审计轨迹。

### 视觉五：一次 Agent 购买算力的完整流程

Agent 请求 GPU / API → 服务返回价格 → Agent 检查预算 → 钱包支付 0.02 USDC → 链上结算 → 服务立即开放 → 交易记录进入企业账本

## 13｜Agent 经济的四阶段推演

### 阶段一：人类出资，Agent 代付

- Agent 使用一次性卡、共享支付令牌或小额稳定币钱包；
- 人类设定明确限额；
- 主要场景是购物、订票、API 和云服务。

### 阶段二：Agent 持有经营预算

- Agent 钱包预存稳定币；
- 自动购买数据、算力、模型调用和软件服务；
- 形成 Working Capital（营运资金）和日内现金管理需求。

### 阶段三：Agent 可以赚钱并支付其他 Agent

- Agent 出售数据、研究、计算、内容和执行服务；
- Agent 之间形成机器对机器市场；
- 支付从一次性消费扩展到持续经营现金流。

### 阶段四：Agent 成为自动资金路由器

Agent 自动比较：

- 哪种稳定币流动性最好；
- 哪条链费用最低；
- 哪个钱包授权最安全；
- 哪个平台奖励最高；
- 哪个支付方式具有退款与争议保护；
- 哪个司法辖区允许交易。

最终可能形成：

> **机器驱动的稳定币最优执行与自动现金管理。**

## 14｜Agent 经济对稳定币商业模式的影响

### 影响一：稳定币从“持有资产”变成“机器工作资金”

Agent 需要预存少量余额，类似企业备用金和云账户余额。

### 影响二：交易量可能大增，但平均余额未必同步增长

- 按次支付提高周转率；
- 即时充值和即时结算减少沉淀；
- 发行人可能获得网络价值，却不一定获得同等储备收入。

### 影响三：Agent 平台可能成为新的超级分销商

- Agent 钱包决定默认稳定币；
- 云平台决定支付协议；
- 模型平台决定服务发现；
- Stripe、Coinbase、Visa、AWS 等可能要求分享支付费和储备收益。

### 影响四：Open USD 模式与 Agent 经济高度匹配

如果 Agent 平台为 OUSD 带来大量采用：

- 平台可分享储备收益；
- OUSD获得机器支付流量；
- Open Standard形成比单一发行人更开放的分销激励；
- Agent可能根据机器可读收益与费用自动选择OUSD。

### 影响五：稳定币竞争将从品牌竞争转向机器可读规则竞争

未来重要字段可能包括：

- 可用链；
- 结算速度；
- 交易费；
- 冻结与合规规则；
- 赎回条件；
- 储备收益分享；
- 白名单和限额；
- 争议解决；
- API 可用性。

## 15｜Agent 支付的风险与控制框架

### 主要风险

- Prompt Injection（提示注入）诱导付款；
- 私钥或权限泄露；
- 不可逆支付；
- Agent 身份与法律代理关系不明确；
- 制裁与反洗钱要求；
- 税务和会计记录；
- 多 Agent 连锁付款；
- 稳定币脱锚、链拥堵和智能合约风险；
- 服务未交付但支付已完成。

### 必须展示的控制措施

- 单笔、日度和类别限额；
- 商户和地址白名单；
- 高风险交易人工确认；
- 可撤销授权；
- 支付用途证明；
- 分层钱包；
- 延迟结算或托管机制；
- 完整审计日志；
- 合规策略包装器；
- 异常行为自动暂停。

# 六、全文最终结论

全文最终应落到六个结论：

1. **稳定币是一张数字化资产负债表，不只是支付代币。**
2. **流通量决定毛收入，分销结构决定净利润。**
3. **Hyperliquid、Coinbase、Open Standard和Agent平台证明，真实场景可以反向要求发行人分享储备收益。**
4. **稳定币正在成为美元的互联网分发层，并增加对美国短期国债的需求，但净新增需求取决于资金来自哪里。**
5. **稳定币主要支持短端美债，不能自动解决美国长期财政融资和久期供给问题。**
6. **Agent经济可能显著提高稳定币支付频率，但真正决定发行人利润的是Agent是否沉淀经营余额，以及谁控制Agent的钱包和支付路由。**

结尾句建议：

> 稳定币把美元搬上区块链；Open USD把储备利润交给分销网络；Agent经济则可能把数字美元交给机器自主选择。下一轮竞争，争夺的不只是发行一枚美元，而是谁成为全球人类与机器默认的结算入口。

- 不把Agent经济作为突兀附录；
- 不把公司逐一罗列而缺乏主线。

## 2. 段落架构

每一章统一使用四层结构：

1. **问题**：本章要回答什么；
2. **机制**：资金和权力如何流动；
3. **案例／数据**：用公司、协议和数字验证；
4. **投资含义**：对公司、资产或市场意味着什么。

长段落拆成：

- 一句核心判断；
- 3至5个因果节点；
- 一张图；
- 一个风险边界。

## 3. 专业度

专业度来自：

- 区分毛收益与净经济价值；
- 区分交易量与平均余额；
- 区分短债需求与长期国债融资；
- 区分支付美元化、储备货币和美债需求；
- 区分支付稳定币、收益型代币和代币化货币基金；
- 区分Open Issuance、Open Standard与Open USD；
- 区分Agent支付量增长与稳定币发行人利润增长。

## 4. 深度

深度不靠堆公司，而靠回答五个结构性问题：

1. 为什么储备收益开始从发行人流向渠道？
2. Bridge为何同时发展多品牌币和共享币？
3. 稳定币对美债究竟是净新增需求还是存量迁移？
4. 美元结算网络强化是否会削弱其他国家货币主权？
5. Agent经济增加的是交易速度、余额，还是平台议价权？

## 5. 通俗易懂

使用三个贯穿全文的类比：

- 稳定币发行人：数字时代的现金管理基金加支付发行人；
- Hyperliquid / Coinbase / Open Standard：大型销售渠道与应用商店；
- Agent钱包：具有预算、权限和自动付款能力的企业备用金账户。

任何复杂机制必须先给一句白话，再给专业定义。

## 6. 风险与事实边界

全文不得：

- 把AQAv2约90%写成Circle全部收入的90%；
- 把稳定币市值直接等同新增美债需求；
- 把稳定币强化美元结算写成美元地位永远不可挑战；
- 把Agent交易量直接等同稳定币储备收入；
- 把Open Stablecoin Alliance与Open Standard / Open USD混为一谈；
- 把Open Issuance与OUSD当作同一产品；
- 把HYPE视为Hyperliquid Labs股权；
- 把支付稳定币称为银行存款；
- 在缺乏合同披露时编造最终利润分配金额。
