# 全球新金融基础设施｜第四篇策划案

## 暂定标题

**AI Agent 的钱包战争**  
**机器如何持有资金、选择稳定币、授权支付，并决定下一代金融入口**

## 与第三篇的分工

第三篇研究稳定币的利润、美元结算和美债需求。第四篇独立研究 Agent 经济，不再与稳定币发行人商业模式混写。

## 核心问题

1. Agent 为什么需要可编程货币和钱包？
2. 卡网络、银行账户、稳定币和代币化存款分别适合哪些机器支付场景？
3. Agent 如何获得预算、权限、身份和审计轨迹？
4. x402、Machine Payments Protocol（机器支付协议）及其他协议如何竞争？
5. Stripe、Coinbase、Visa、Mastercard、PayPal、银行、云平台和模型平台谁控制默认入口？
6. Agent 支付增加的是交易量、资金周转率还是沉淀余额？
7. 谁最终获得支付费、稳定币分润、钱包费和订单路由价值？

## 建议结构

1. 人类支付系统为何不适合自主 Agent
2. Agent 钱包的六层技术栈
3. 卡支付与稳定币支付的场景分工
4. x402、MPP 与机器可读支付条件
5. Agent 预算、授权和企业内部控制
6. Agent 营运资金与自动现金管理
7. 机器对机器市场
8. 默认稳定币和默认钱包战争
9. Stripe、Coinbase、Visa、AWS、模型平台与银行竞争图
10. 安全、合规、税务和责任边界
11. 四阶段发展推演
12. 投资映射与监测指标

## 四阶段推演

- 阶段一：人类出资，Agent 代付。
- 阶段二：Agent 持有小额经营预算。
- 阶段三：Agent 赚取收入并向其他 Agent 支付。
- 阶段四：Agent 自动比较稳定币、链、支付协议、退款能力和司法辖区，形成自动资金路由。

## 核心区分

- Stablecoin Volume（稳定币交易量）
- Stablecoin Velocity（稳定币周转速度）
- Stablecoin Float（稳定币沉淀余额）

Agent 支付频率增长并不自动等于稳定币发行人储备收入增长。发行人需要 Agent 沉淀经营余额；支付平台则可能从每次路由、授权和服务调用中获利。

## 风险控制

- 单笔、日度和类别限额
- 商户和地址白名单
- 高风险交易人工确认
- 分层钱包与可撤销授权
- Prompt Injection（提示注入）防护
- 制裁与反洗钱策略
- 交易用途证明与完整审计日志
- 服务未交付时的托管或争议处理
