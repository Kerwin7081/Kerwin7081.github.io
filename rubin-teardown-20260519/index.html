引言 · 推文缘起

Vera 交付 SpaceX：这是一张 AI 基础设施分层重估的起点图

用户提供的截图显示，NVIDIA AI Infrastructure 账号表示，合作伙伴 SpaceX 正在试用 NVIDIA Vera CPU，并称 Vera 是 NVIDIA 为代理式 AI 打造的 CPU。这个信息本身很短，背后指向一个更大的产业变化：AI 基础设施的焦点，正在从单颗 GPU 性能扩展到整机柜、整集群、整 AI Factory 的系统吞吐。

过去市场讨论 AI 硬件，最容易聚焦在 GPU、HBM、CoWoS 与电力。Rubin 这一代更值得注意的是，它把 Vera CPU、Rubin GPU、NVLink 6、ConnectX-9、BlueField-4、Spectrum-X 以及 CMX/STX 上下文存储放进同一个系统叙事。我的理解是，NVIDIA 正在告诉市场：Agent 时代是整套计算、记忆、调度、网络和存储的协同系统时代。

核心判断

Rubin 的投资含义不应只停留在 GPU 更强。它同时暗示三条线：第一，CPU 在 Agent 时代重新变成关键资源；第二，HBM4 继续是 GPU 有效算力的核心瓶颈；第三，CMX/STX 把 eSSD / 3D NAND 从后台存储推向推理热路径。

01 · 架构

Rubin NVL72 是 rack-scale AI supercomputer，不是 72 张显卡插满机柜

NVIDIA 官方对 Vera Rubin NVL72 的定义是：72 颗 Rubin GPU、36 颗 Vera CPU、ConnectX-9 SuperNIC、BlueField-4 DPU、NVLink 6 switch，并通过 Quantum-X800 InfiniBand 与 Spectrum-X Ethernet 对外扩展。这个定义很关键，它说明 Rubin 是机柜级协同系统，而不是单个 GPU 板卡的简单堆叠。

从性能和内存结构看，NVL72 整柜提供 3.6 exaFLOPS NVFP4 inference、2.52 exaFLOPS NVFP4 training、54TB LPDDR5X CPU 内存、20.7TB HBM4 GPU 显存。单颗 Rubin GPU 对应 288GB HBM4，单颗 Vera CPU 可支持最高 1.5TB LPDDR5X，并通过 NVLink-C2C 与 GPU 形成高带宽协同。

02 · 定价

一个 Rubin 机柜多少钱？中枢先用 600 万美元做测算

NVIDIA 没有公布 Vera Rubin NVL72 的正式 list price。供应链报道给出的 VR200 口径是，Vera Rubin-based NVL72 系统当前报价约 500 万到 700 万美元，并提示这些报价可能只是 ODM 价格，不一定包含完整质保和服务。因此，本文采用 600 万美元一柜作为中枢估算，用于拆解价值分布，而不是声称这是官方定价。

这个价格级别的意义在于，它已经不是普通服务器采购概念，而是 AI 工厂资本开支单元。如果客户采购的是多柜 SuperPOD 或更大规模 AI Factory，Rubin 的单柜 BOM 结构会逐层传导到 HBM、NAND/eSSD、液冷、电力、网络、PCB、连接器和系统集成商。

价格口径说明

500万到700万美元来自供应链报道，不是 NVIDIA 官方确认价格。后续真实成交会受配置、服务、质保、交付批次、客户等级、网络与存储配套、数据中心改造成本影响。

03 · BOM

拆一个 Rubin 机柜：最大头仍是 GPU + HBM，但 CMX/eSSD 已经成为新变量

以 600 万美元为中枢售价估算，Rubin NVL72 的核心价值仍集中在 GPU 模组。上方饼图采用非重复分摊口径，把 HBM4 从 GPU 模组中单独拆出，便于观察部件权重。这里的 GPU 模组不是裸芯片，而是 Rubin GPU、HBM4、先进封装、基板、供电、互联和板级工程的组合。

单独把 HBM4 拆出来看，整柜约 20.7TB HBM4，如果按 35 到 60 美元/GB 粗估，对应约 72 万到 124 万美元级别。这解释了为什么 HBM4 在 Rubin 世代仍是最核心的供给约束。

第二个更容易被市场忽略的部分，是 3D NAND / eSSD / CMX。供应链报道提到 VR200 系统 come with about $1 million of 3D NAND storage。我的理解是，这更像随 Rubin AI 系统配置的 CMX / STX 上下文存储层，可能以 storage server、storage node 或 pod-level flash pool 的形式存在，并不一定是塞在 NVL72 compute tray 内部的普通本地盘。

04 · CPU 回归

72 GPU 对 36 CPU：Rubin 暗示 CPU 在 Agent 时代重新变成瓶颈资源

Rubin NVL72 的配置是 72 颗 Rubin GPU 对 36 颗 Vera CPU，即 2 GPU : 1 CPU。这个比例已经比过去很多 1 颗 CPU 带 4 到 8 张 GPU 的传统 GPU 服务器更重视 CPU。更重要的是，NVIDIA 官方对 Vera CPU 的定位不是普通主机 CPU，而是 purpose-built for data movement and agentic reasoning。

Agentic AI 不是一次输入、一次推理、一次输出。它包含任务拆解、工具调用、浏览器环境、代码执行、API 调用、权限控制、数据库检索、RAG、KV cache 管理、多轮状态维护。这些负载很多不是 GPU 矩阵乘法，而是 CPU 侧环境计算、控制流、数据流和系统编排。

Chatbot 时代：Prompt 输入 → GPU 推理 → 输出答案
↓
Agent 时代：任务规划 → 工具调用 → 环境执行 → 检索与记忆 → 多轮验证 → GPU/CPU/存储反复协同
↓
结论：GPU 决定算力峰值，CPU 决定 Agent 执行效率与系统吞吐稳定性
05 · CMX / STX / eSSD

约 100 万美元 3D NAND 存储的真正含义：NAND 开始进入推理热路径

CMX 是 NVIDIA 在 STX 架构下的 context memory storage platform。NVIDIA 官方将 CMX 定义为 AI-native context tier，用于 long-context、multi-turn、agentic AI inference；它由 BlueField-4 storage processor 驱动，扩展 GPU memory，形成 shared pod-level context tier，并针对临时 KV cache 优化。

这句话对 NAND/eSSD 非常重要。过去 SSD 更多被看作后台容量，负责存训练数据、日志、对象存储缓存和企业文件。但在 Agentic inference 里，KV cache、RAG 向量索引、企业知识库、工具调用状态、多轮任务上下文，会变成频繁访问、延迟敏感、容量远大于 HBM 的数据层。HBM 放不下，普通存储太慢，于是 eSSD / 3D NAND 被推向中间层。

图：NVIDIA CMX Context Memory Storage Platform。CMX 更像 pod-level context memory tier，服务 KV cache、长上下文与多轮 Agent 工作流。

因此，Rubin 对 eSSD 的意义不是 eSSD 替代 HBM。更准确的表达是：HBM 仍是 GPU 第一内存层，eSSD/CMX 承接 HBM 与 DRAM 放不下、传统存储又太慢的上下文外溢层。这个层级一旦影响 tokens per second、GPU 利用率和每百万 token 成本，NAND 的估值逻辑就会从普通周期品向 AI 基础设施部件迁移。

铠侠是不是供应商？

铠侠是明确参与者之一，但不能说 CMX/STX 的 eSSD 供应商只有铠侠。公开资料显示，铠侠围绕 NVIDIA Storage-Next / GPU-accessible memory 推出 GP Series / Super High IOPS SSD 方向，目标是用 XL-FLASH、低延迟、高 IOPS、GPU 直连来补 HBM 容量不足。与此同时，Samsung、Micron、Solidigm、Phison 等也会在企业级 SSD、QLC eSSD、控制器、系统认证和 AI storage 方案里参与竞争。

06 · 投资推演

Rubin 的真正信号：AI Infra 从买 GPU 进入买整套工厂能力

Rubin 系统最重要的投资含义，是价值分配从单一 GPU 芯片扩散到整柜系统。NVIDIA 仍然是最大赢家，因为它同时控制 GPU、CPU、NVLink、DPU、NIC、软件栈和越来越多的整柜交付环节。但价值不会只停留在 NVIDIA。HBM、eSSD、液冷、电源、网络、retimer、连接器、PCB、ODM 集成都会被 Rubin 这种系统级产品重新定价。

01
NVIDIA 平台化能力继续增强

如果 NVIDIA 交付更完整的 L10 compute tray，ODM 的角色会从系统设计者更偏向机柜集成者，价值继续向 NVIDIA 和关键半导体供应链集中。

02
CPU 需求回归不是传统服务器周期

Vera CPU 的意义在于 Agent 环境、工具、调度、KV cache 管理和数据流动。CPU/GPU 配比变化是 AI 工作负载结构变化的硬件结果。

03
HBM 仍是第一瓶颈

20.7TB HBM4 是 Rubin NVL72 的关键价值池。HBM 对 GPU 算力释放的约束仍会强化存储龙头的议价能力。

04
eSSD / NAND 成为第二层记忆基础设施

CMX/STX 说明长上下文、RAG、KV cache 与 Agent 状态需要新的高速存储层。NAND 不再只是消费电子和普通服务器周期品。

Kerwin 观点

我认为 Rubin 对市场最大的提醒是：AI 基础设施不能再被简化成 GPU 缺货。未来的投资框架要从 GPU 一层扩展到 GPU + HBM + CPU + LPDDR/DRAM + CMX/eSSD + 网络 + DPU + 液冷 + 电力的系统性再定价。HBM 是第一内存层，CMX/eSSD 是外置上下文记忆层，两者共同构成 Agentic AI 的新硬件底座。

结语：Rubin 是 AI 工厂的剖面图

Vera 交付 SpaceX 的推文，是一个非常好的观察切口。它让市场重新看到，Agentic AI 需要更多 GPU，也需要更多 CPU、更大的 HBM、更快的上下文存储、更强的 DPU/NIC、更复杂的网络、更高密度的液冷和电力系统。

拆解 Rubin 后，我的结论很明确：GPU 仍然是 AI Infra 的核心资产，HBM 仍然是第一瓶颈，但 CPU 与 CMX/eSSD 正在成为新的边际变化。谁能理解这一层变化，谁就能更早看懂 AI 基础设施从芯片周期走向工厂系统周期的投资机会。
