Thesis · Kerwin 洞察

Agent 是 AI 从“会回答”走向“能交付”的关键层

2026 年 Agent 产业最重要的变化，不只是模型能力继续提升，而是 Agent 开始被放进操作系统、办公软件、超级 App、电商后台、企业协作与本地 PC。它不再只是开发者社区里的自动化实验，而是开始成为 AI 生产力工具的分发形态。

Kerwin 的核心判断是：AI 真正进入企业与个人生产率系统的决定性检验，不是模型参数继续变大，而是 Agent 能否被安全、低门槛、持续地嵌入真实工作流。

Kerwin Insight

Agent 可能成为 AI 时代的新应用层。

PC 时代，软件承载生产力。手机时代，App 承载服务与消费。微信生态里，小程序承载轻量交易与生活服务。AI 时代，Agent 可能承载任务执行、企业流程和个人生产力。

区别在于，软件、App、小程序主要等待用户主动操作；Agent 在用户授权后，可以持续理解目标、拆解任务、调用工具、访问数据并执行动作。

图解：Agent 的意义在于把 AI 从回答界面推进到任务执行界面。

01 · Windows × OpenClaw

Microsoft 把 OpenClaw 放进工作入口，Agent 从社区工具进入主流办公生态

Microsoft Build 2026 的关键信号，是 Microsoft Scout 被定义为 built on OpenClaw 的 always-on personal agent for work。它运行在 Teams、Outlook、OneDrive、SharePoint、本地设备动作等工作场景中，并使用 Work IQ 作为上下文引擎。

同时，Windows 已经开始支持 OpenClaw on Windows preview。Microsoft Execution Containers 提供操作系统级的 identity、isolation、governance 与 policy enforcement，让 Agent 可以在本地执行多步骤工作流，同时处于系统强制边界内。

产业含义

过去企业不敢部署 Agent，核心原因是不会搭建、不敢授权、难以审计。Windows × OpenClaw × MXC × NVIDIA OpenShell 的组合，正在把 Agent 从“开发者自己折腾”变成“企业可以管理的生产力工具”。

组件解决的问题对 Agent 普及的意义
OpenClawAgent 框架、技能、任务执行与持续运行让开发者和平台可以围绕同一类框架构建任务型 AI
Microsoft Scout常驻工作型 Agent进入 Teams、Outlook、OneDrive、SharePoint 等核心办公入口
MXC本地执行隔离、身份、策略和治理降低企业 IT 与合规部门的采用阻力
NVIDIA OpenShellAgent 沙箱运行时、策略、推理路由、隐私保护把本地 Agent 的安全运行与 NVIDIA AI 栈绑定
RTX Spark本地算力、统一内存、个人 Agent PC让 Agent 从云端调用延伸到本地执行节点
图解：Agent 的平台化需要入口、框架、安全运行时和本地算力同步成熟。
02 · NVIDIA 五层理论

NVIDIA 的真实目标：把 AI Factory 与 Agent 应用层连接起来

如果只看 GPU，容易低估 NVIDIA 的战略野心。它正在用硬件、系统、软件、模型和 Agent 五层，把 AI 基础设施与企业生产率工具打通。

Agent 是第五层，但它会反向拉动前四层。Agent 越多，推理调用越多，数据中心利用率越高，对 GPU、CPU、DPU、网络、存储、电力、液冷和安全运行时的需求越强。

NVIDIA Agent 五层闭环
第五层
Agent 应用层：OpenClaw、NemoClaw、Microsoft Scout、企业 Agent、电商 Agent、社交 Agent。
第四层
模型与技能层：Nemotron、CUDA-X skills、领域模型、企业数据连接器。
第三层
系统软件与运行时：CUDA、NIM、NeMo、OpenShell、Agent Toolkit。
第二层
计算硬件与系统：Rubin GPU、Vera CPU、BlueField、Spectrum、NVLink、RTX Spark。
第一层
AI Factory 基础设施：电力、液冷、机柜、网络、存储、调度、运维。
投顾视角

NVIDIA 卖的不再只是 GPU，而是一套从 AI Factory 到 Agent 应用层的生产率基础设施。它的估值逻辑，也因此从半导体周期进一步扩展到平台绑定、工作流渗透和持续推理需求。

03 · 平台入口竞争

超级入口正在把 Agent 从技术概念推向大众分发

Agent 的普及不完全取决于普通用户是否愿意自己搭建。真正的拐点，是超级入口开始预装、内置、分发和管理 Agent。

Microsoft 把 Scout 放进工作入口，Tencent 让 WeChat 接入 OpenClaw，Alibaba 推出企业 Agent 平台 Wukong，Meta 推出覆盖 WhatsApp、Messenger、Instagram 的 Business Agent。这些动作共同指向一个趋势：Agent 正在从开发者驱动转向平台分发驱动。

Microsoft

Scout 进入 Teams、Outlook、OneDrive、SharePoint 和本地设备动作。核心价值是企业工作流。

NVIDIA

OpenShell、RTX Spark、Nemotron、CUDA-X 把 Agent 与本地算力、推理和专业技能库绑定。

Tencent

WeChat 与 OpenClaw 的结合，意味着 Agent 有机会进入通信、支付、服务、小程序和商业闭环。

Alibaba

Wukong 面向企业任务协调，Qwen 生态延伸到电商、协作和商家运营，适合 B 端工作流。

Meta

Business Agent 进入 WhatsApp、Messenger 和 Instagram，瞄准客服、销售、预约和线索筛选。

Developers

OpenClaw、Hermes、Manus 等框架把 Agent 从模型调用扩展为技能、记忆、工具和持续执行。

Kerwin Insight

如果 Windows、Microsoft 365、WeChat、Taobao、WhatsApp、Instagram 等超级入口持续内置 Agent，Agent 的采用曲线可能从开发者驱动切换为平台分发驱动。

这会显著降低采用门槛。个人用户不需要自己部署，商家不需要自己写工具链，中小企业不需要从零训练模型，开发者也不需要独自处理权限、安全、沙箱和运维。

04 · 产业链传导

Agent 普及会反向拉动 AI Factory 与企业软件重构

Agent 的商业价值在应用层体现，但它的成本和能力由底层基础设施决定。一个真正可用的 Agent，需要持续推理、多轮调用、长上下文、工具执行、本地文件访问、企业权限和审计能力。

第一步：入口内置
Windows、M365、微信、淘宝、WhatsApp 等入口把 Agent 放到用户和企业日常工作流中。
第二步：任务执行
Agent 从聊天助手升级为文件整理、会议纪要、销售线索、客服分流、代码生成、数据分析和电商导购。
第三步：持续推理
任务越复杂，越依赖持续推理、长上下文、工具调用、低延迟网络、存储和本地算力。
第四步：基础设施扩容
GPU、CPU、DPU、NIC、交换机、HBM、存储、电力、液冷和安全运行时同步受益。
链条受益逻辑代表方向
平台中枢控制入口、运行时、模型、推理和生态NVDA、MSFT、META、BABA、TCEHY
推理基础设施Agent 增加持续推理与多工具调用GPU、ASIC、HBM、DPU、网络、存储
企业软件高价值工作流被 Agent 自动化EDA、工业软件、协作软件、CRM、客服系统
端侧与 PC本地 Agent 需要更强 NPU、GPU、内存和安全沙箱AI PC、工作站、RTX Spark、Windows 设备生态
安全治理Agent 有身份、有权限、有行为轨迹，必须可审计身份治理、零信任、终端安全、数据治理
05 · 投资映射

从投顾视角看，Agent 主题要分层，而不是简单买概念

Agent 主题的难点在于，很多公司都可以讲故事，但绑定度和盈利路径差异很大。更合理的框架，是把它拆成平台中枢、专业工作流、推理基础设施、端侧 PC、安全治理五个层级。

一、平台中枢

NVDA、MSFT。重点看 Agent runtime、Windows 分发、本地算力、持续推理和企业安全栈。

二、超级入口

META、BABA、TCEHY。重点看用户入口、商家工具、交易闭环与企业服务订阅。

三、专业工作流

SNPS、CDNS、Siemens、Dassault。重点看 EDA、工业仿真、设计验证和数字孪生是否 Agent 化。

四、推理基础设施

AVGO、MRVL、ANET、MU、Samsung、SK Hynix、COHR、LITE、AAOI、SIVE。重点看网络、HBM、光通信和 CPO。

五、AI PC 与端侧

DELL、HPQ、Lenovo、AMD、INTC、QCOM、MediaTek。重点看本地 Agent 是否推动硬件规格升级。

六、安全治理

CRWD、PANW、ZS、OKTA、PLTR。重点看身份、权限、数据治理、审计和 Agent 行为可控性。

投资判断

Agent 不是一个单一赛道，而是一种新的应用分发形态。真正受益的公司，通常具备三类条件：控制入口，控制运行时，或者控制高价值工作流。仅仅贴上 Agent 标签，无法构成足够强的投资逻辑。

06 · 风险约束

OpenClaw 可能是重要底座，但还不能直接定义为唯一标准

OpenClaw 正在成为重要的 Agent 框架，尤其是在 Windows 和 WeChat 等入口加入后，它的产业地位明显上升。但从投资研究角度，仍需保持三点克制。

风险解释观察指标
标准多元化OpenAI、Anthropic、Google、Meta、Alibaba、Tencent 都会推动自己的 Agent 体系Windows、云平台、超级 App 最终支持哪些框架
安全风险Agent 访问文件、邮件、支付、代码库和企业系统后，攻击面明显扩大沙箱、权限、审计、提示注入防护是否成熟
ROI 验证企业采用需要证明能降本、增效、减少错误或提升收入转化企业续费率、使用频次、任务完成率、人工替代率
估值提前反映平台型公司估值已经包含大量 AI 预期收入兑现速度与资本开支回报率
克制结论

更稳妥的表述是：OpenClaw 代表的 Agent 框架正在从开发者工具升级为生产力底座的重要候选。它不必成为唯一标准，只要被 Windows、WeChat、企业软件和本地 PC 生态广泛支持，就足以推动 Agent 进入平台化扩散阶段。

结语：AI 投资主线正在从算力建设走向生产力兑现

2023 到 2025 年，AI 投资主线主要围绕训练大模型、GPU 扩容和 AI 数据中心建设展开。进入 2026 年后，市场需要回答更现实的问题：企业如何把 AI 变成收入、效率和流程改造。

Agent 正是这条链的兑现层。它把模型能力、数据权限、工具调用、系统入口和安全运行时组合起来，成为可执行的工作单元。Windows 与 OpenClaw 的合作，NVIDIA OpenShell 与 RTX Spark 的推出，以及 Tencent、Alibaba、Meta 的平台化动作，共同说明 Agent 正在进入主流分发阶段。

Kerwin 的最终判断：AI 时代的新应用层，很可能不是一个个静态 App，而是一个个可以被授权、被治理、被审计、能持续完成任务的 Agent。谁掌握 Agent 的入口、运行时、安全机制和工具生态，谁就更接近掌握下一代 AI 生产力底座。
