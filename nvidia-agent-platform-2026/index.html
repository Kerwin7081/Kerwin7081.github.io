Microsoft Build 2026 · 重大合作

Windows × OpenClaw 正式合作

核心事件

2026年6月2日，Microsoft Build 2026 大会上，微软宣布：
1. OpenClaw on Windows 进入 Preview——OpenClaw 节点和网关通过 MXC（Microsoft Execution Containers）在 Windows 上安全运行。

3. NVIDIA OpenShell 登陆 Windows——基于 MXC 构建，内核级安全沙箱。

MXC：Windows 的 AI Agent 沙箱

Microsoft 为 AI Agent 专门设计的 Execution Containers（MXC） 是此次合作的技术底座。它提供 OS 级别的 Agent 执行隔离：代码执行、文件访问、网络交互——全部在沙箱内受限运行。结合 Defender、Entra、Intune、Purview 等企业安全产品，IT 团队可以实现对本地 Agent 的全面约束和审计。

产品落地 · OpenClaw 最大商业化信号

Microsoft Scout：由 OpenClaw 驱动的常驻个人 Agent

重磅

Microsoft Scout 是微软在 Build 2026 发布的旗舰级个人 AI Agent 产品，定位为"work 场景的 always-on personal agent"。

底层技术栈：OpenClaw 开源项目。微软官方原话——"It is powered by OpenClaw open-source technology, reflecting our commitment to building with the community while extending capabilities for the enterprise."

Scout 的定位与能力

01
常驻运行，非一问一答

Scout 是一个 Claw 类型的 Agent，持续在后台运行，自主执行任务、适应障碍、主动汇报。不像 Copilot 那样等待用户提问。

02
企业级安全管控

内置 Defender、Entra、Intune、Purview 保护。运行在 MXC 隔离沙箱内。IT 管理员可以配置文件/网络/应用的访问策略。

03
跨 Microsoft 365 深度集成

内置连接 Outlook、Teams、SharePoint、OneDrive 等 M365 生态。可以跨应用调度任务、汇总信息、生成报告。

04
技能可扩展（Agent Skills）

开发者可以向 Scout 添加自定义技能（Skills），扩展其能力边界。与 NemoClaw 蓝图和 CUDA-X 技能库兼容。

对 OpenClaw 生态的里程碑意义

这是开源 AI Agent 框架首次被 Microsoft 作为旗舰级个人 Agent 产品的底层引擎。
这意味着：① OpenClaw 从社区项目进入企业级生产环境 ② 微软背书等于数亿 Windows 用户的潜在分发渠道 ③ NemoClaw + OpenShell + Windows MXC = 企业 Agent 标准栈初步成形。

结语

NVIDIA 的 Agent 平台战略是一个设计精巧的纵向锁定：用免费的 NemoClaw Blueprints 降低开发门槛 → 用 OpenShell 建立安全信任 → 用 Nemotron 模型提供推理引擎 → 最终让企业客户的所有 Agent 工作负载绑定在 NVIDIA 的硬件-系统-软件-模型四层基础设施之上。

OpenClaw 被微软选为 Scout 的底层引擎，标志着开源 Agent 框架正式进入亿级用户分发渠道。

最重要的判断：Agent 不是 NVIDIA 的副业，而是 Jensen Huang 明确宣告的第五层——即五层战略的最终交付物。这意味着未来 NVIDIA 的 CapEx、并购、生态合作和产品路线图，都将围绕 Agent 层做纵向整合。
