02 · THE PRECISE MAPPING

关键校准：Virgo 并不直接等于 NVLink

判断 1.6T、CPO 和光模块需求时，必须把 Scale-up、Scale-out 与 Scale-across 分开。

CPU
Google AxionArm 通用 CPU，承担 Agent Sandbox、工具调用、微服务和数据处理。
VS
NVIDIA Vera CPU围绕 AI 工厂的数据移动、Agent 逻辑与 Rubin 机架协同设计。
加速器
TPU 8t / 8i8t 偏训练，8i 偏推理和强化学习，重点是性能功耗比与成本控制。
VS
Rubin GPU通用 AI 加速器，CUDA 生态成熟，并与 HBM4、NVLink 深度整合。
Scale-up
TPU ICI + OCS芯片、Cube、Pod 与 Superpod 内部紧耦合互联，更接近 NVLink/NVSwitch。
≈
NVLink 6 + NVLink Switch让 72 颗 Rubin GPU 在单机架内形成高带宽计算域。
Scale-out
Virgo NetworkPod 和机架之间的专用 RDMA 网络，采用扁平两层、高端口密度与非阻塞设计。
≈
Spectrum-X / Quantum-X把多个 NVL72 机架扩展为 SuperPOD，承担集群横向通信。
Scale-across
Virgo + Pathways/JAX把多个园区组织为逻辑训练集群，突破单栋建筑的电力和空间上限。
≈
Spectrum-XGS让相距数百公里的数据中心作为统一 AI 环境运行。
软件
JAX / XLA / PyTorch / vLLM降低客户迁移到 TPU 的成本，并保留 NVIDIA A5X 选择。
VS
CUDA / NCCL / Dynamo成熟度、开发者生态和整机验证构成最深护城河。
技术结论

机架内部互联中，TPU ICI 更接近 NVLink；机架之外和跨园区扩展中，Virgo 更接近 Spectrum-X、Quantum-X 与 Spectrum-XGS。只有完成这一层级校准，才能判断 1.6T 和 CPO 的价值出现在哪里。

比较维度Google 路线NVIDIA 路线投资含义
系统形态TPU、Axion 与 NVIDIA GPU 并存高度整合的参考架构与整机系统Google 强调选择权和成本；NVIDIA 强调性能确定性
商业目标提升 Cloud 利用率、毛利率与客户黏性扩大 GPU 之外的 CPU、网络、DPU 和软件收入双方均在争夺系统级价值
竞争关系以 TPU 替代部分 GPU，同时销售 A5X Rubin把 Google 当客户，也面对 TPU 竞争典型的客户兼竞争者

04 · WHY VIRGO MATTERS

Virgo：把“数据中心园区”变成一台计算机

大型训练已经逼近单个数据中心的电力、空间和散热上限。Virgo 通过高端口密度交换、扁平两层非阻塞拓扑和多平面故障隔离，把多个 TPU Superpod 或 Rubin 机架组织成更大的东西向计算域。

Scale-up Domain｜TPU ICI 与 OCS
连接芯片、Cube 和 Superpod，追求最低延迟与最高紧耦合带宽。
↓
Virgo Scale-out Fabric
连接 Pod 和机架，支持约 134,000 颗 TPU 与 47 Pb/s 非阻塞双向剖分带宽。
↓
Jupiter Front-end
连接通用计算、全球 WAN 与存储，负责数据输入、检查点和服务访问。
↓
Scale-across｜Pathways 与 JAX
把多个 Virgo fabric 连接为超过 100 万颗 TPU 的逻辑训练集群，并计划支持 A5X Vera Rubin。
投资含义

当训练域从单机架扩展到多个数据中心，连接成为决定 GPU/TPU 利用率的“计算乘数”。网络减少拥塞、尾延迟或故障恢复时间，都可能直接提高昂贵加速器的有效产出。

05 · FROM VIRGO TO 1.6T

为什么 Virgo 会自然连接到 1.6T 与 CPO

Virgo 没有公开披露具体光模块供应商，但其网络形态会扩大高速光连接需求。随着交换芯片从 51.2T 迈向 102.4T，端口速率从 400G、800G 继续升级至 1.6T，并依靠每通道 200G PAM4 降低端口和布线复杂度。

400G｜上一代主力
4×100G 或 8×50G，广泛用于云数据中心前后端网络。
800G｜当前放量核心
8×100G 或 4×200G，AI Scale-out 集群带动需求快速增长。
1.6T｜Rubin 与 102.4T 时代
典型为 8×200G，提升端口密度并服务更大规模机架间网络。
3.2T｜下一阶段
400G/lane 与 204.8T 交换平台的潜在方向，目前仍处于更早阶段。

1.6T 可插拔模块

模块位于交换机前面板，可独立更换，生态成熟、维护方便，是近期最现实的部署路径。

  • 价值：光引擎、DSP、EML/硅光、封装与测试
  • 优势：可维护性、标准化和供应链灵活
  • 挑战：功耗、面板密度和高速电通道损耗

Co-Packaged Optics

把光引擎放到交换 ASIC 附近，缩短高速电连接，降低功耗并提高链路稳定性。

  • 价值：光引擎、硅光、外置激光源与光纤阵列
  • 优势：功耗、密度和可靠性
  • 挑战：维修、热管理、良率和供应链重构
核心判断

1.6T 可插拔与 CPO 并非二选一。未来较长时间内，两者会按照距离、机架密度、维护要求和客户偏好并存。

结语：连接把芯片变成系统

Google 拥有 Axion、TPU、ICI、Virgo、JAX 和全球云基础设施；NVIDIA 拥有 Vera、Rubin、NVLink、Spectrum-X、BlueField 和 CUDA。两家公司正从不同方向抵达同一终点：把数万乃至百万颗处理器组织成稳定、高利用率、可商业化的 AI 工厂。

Kakao 证明 TPU 开始具备从内部工具向外部平台扩散的现实基础；Virgo 则把竞争继续推向交换芯片、NIC、1.6T、CPO、存储、电力和液冷。中际旭创代表的是全球 AI 带宽升级的结构性映射，不能被简单写成 Google 供应商故事。
